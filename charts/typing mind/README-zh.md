## 👋 概述

Typing Mind 是一个 Chat UI 前端，允许你使用所有主流 AI 模型（如 ChatGPT、Claude、Azure OpenAI、Gemini 等）并结合你自己的 API 密钥进行交互。

通过使用类似 Typing Mind 的 Chat UI 前端和 API 密钥，你可以获得以下优势：

- 你只需为你实际使用的部分付费（而不是像 ChatGPT Plus 那样的月度订阅）
- 你的聊天记录不会被 OpenAI / Claude 等用于训练模型
- 在统一界面中便捷地与多个模型交互
- 享受专业、先进的聊天界面和丰富的功能

Typing Mind 是**离线优先**且**默认私密**的。你所有的数据、消息和 API 密钥都存储在本地设备上，除了你自己，没有人可以看到。

这是 Typing Mind 的 _静态自托管版本_（[www.typingmind.com](https://www.typingmind.com/)）。通过运行静态自托管版本，你可以从私有服务器或本地设备上使用 Typing Mind。Typing Mind 免费提供基础功能，要解锁所有高级功能需要购买许可证密钥。更多信息请参阅我们的 [定价计划](https://www.typingmind.com/pricing)。

## 🚀 快速开始

要在本地设备或私有服务器上运行该应用：

1. 克隆仓库
2. 安装依赖：`yarn install` 或 `npm install`
3. 启动服务器：`yarn start` 或 `npm run start`
4. 默认情况下应用将在 `localhost:3000` 上运行。

你也可以使用任何其他静态 Web 服务器来托管 `src` 文件夹。

**注意**：如果你在 `localhost` 以外的主机名上运行应用，必须使用 HTTPS 才能确保所有功能正常运行。

## 🔗 连接本地模型

1. 打开应用
2. 点击模型下拉菜单 → 模型设置 → 添加自定义模型
3. 输入你的自定义模型信息：端点、模型 ID。支持自定义请求头和请求体参数。
4. 点击 "Test" 然后点击 "Add Model"。

如果你遇到 CORS 问题，请确保你的本地模型运行在启用 CORS 的模式下。

## ❓ 常见问题

<details>
<summary>
<b>什么是自托管？</b>
</summary>

自托管意味着你可以将 TypingMind.com 上的相同版本部署在你自己的 Web 托管服务器和域名上，供个人使用。
</details>

<details>
<summary>
<b>为什么要自托管？有什么好处？</b>
</summary>

有些人出于隐私和可用性的考虑，更倾向于在自己的服务器上运行软件。例如，如果未来 TypingMind.com 无法访问，你仍然可以从自己的服务器上使用你部署的 TypingMind。自托管版本也可以像本地应用程序一样运行。
</details>

<details>
<summary>
<b>在自托管版本中我可以自定义什么内容？</b>
</summary>

目前，你无法自定义任何内容。自托管版本与你在 TypingMind.com 上使用的版本完全相同。如果你希望使用更灵活、可定制的 Typing Mind，请访问 <a href="https://custom.typingmind.com">https://custom.typingmind.com</a>。
</details>

<details>
<summary>
<b>我可以在哪里部署我的自托管版本？</b>
</summary>

任何地方都可以！你可以部署在自己的服务器上，或者部署在 GitHub Pages、Cloudflare Pages、AWS S3、Vercel、Netlify、Heroku 等静态 Web 云服务上。你甚至可以在本地运行它。
</details>

<details>
<summary>
<b>如果我要自托管，我能获得完整的源代码吗？</b>
</summary>

不能。许可证密钥仅授予你使用和部署该应用到你自己的服务器上的权限。你没有权限修改或重新分发代码。由于该应用不是开源的，因此完整源代码不出售。你只会获得编译后的代码，你可以将其部署到任何地方，而无需修改任何代码或设置。
</details>

<details>
<summary>
<b>如何获取自托管版本的更新？</b>
</summary>

只需从 GitHub 仓库（https://github.com/TypingMind/typingmind）克隆或拉取最新更改，然后重启你的服务器即可。
</details>

<details>
<summary>
<b>使用自托管版本是否仍需要许可证密钥和 OpenAI API 密钥？</b>
</summary>

是的。两者都需要，因为自托管版本与你在 TypingMind.com 上看到的版本完全相同。在自托管版本中也需要许可证密钥。当你输入许可证密钥时，应用会连接到 TypingMind 的许可证服务器以验证你的许可证。
</details>

<details>
<summary>
<b>我可以让我团队/社区/客户使用自托管版本吗？</b>
</summary>

可以，但请注意，他们也需要拥有 TypingMind 许可证密钥和 OpenAI API 密钥才能使用。如果你希望为团队使用 Typing Mind，请查看 <a href="https://custom.typingmind.com">https://custom.typingmind.com</a>。
</details>

<details>
<summary>
<b>我可以将我的许可证密钥/API 密钥嵌入到自托管版本中，让我的团队/社区/客户使用吗？</b>
</summary>

不可以，也不应该这样做。将你的许可证密钥和 API 密钥公开共享是不安全的。静态自托管版本仅用于个人使用。如果你希望为你的团队/社区/客户部署定制版本的 TypingMind，请查看 <a href="https://typingmind.com/custom">https://typingmind.com/custom</a>。
</details>

<details>
<summary>
<b>如果我自托管，能否获得技术支持？</b>
</summary>

自托管版本在你购买许可证密钥后免费提供。如果你自托管，则不提供技术支持。如果你不具备部署自托管版本的技术能力，我们建议你使用托管版本：https://typingmind.com。你也可以查看 Typing Mind Custom 以获得更简单的部署方式和团队管理功能：<a href="https://custom.typingmind.com/">https://custom.typingmind.com</a>
</details>

<details>
<summary>
<b>使用自托管代码的权限有哪些？</b>
</summary>

<ul>
<li>✅ 你可以访问应用的编译代码。</li>
<li>✅ 你有权在自己的服务器上部署和使用编译代码。</li>
<li>❌ 你无权修改或重新分发编译代码。</li>
<li>❌ 你无权共享或转售编译代码。</li>
</ul>
</details>
